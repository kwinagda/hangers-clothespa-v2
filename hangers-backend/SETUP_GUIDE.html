1
Install Required Software
One-time setup on your computer — takes ~15 min
⬇️

Install Node.js (if not installed)

Go to nodejs.org → Download the LTS version → Install it. After installing, open Command Prompt (Windows) or Terminal (Mac) and run:

TERMINAL
node --version # should show v18 or higher npm --version # should show 9 or higher
🐘

Install PostgreSQL Database

Go to postgresql.org/download → Download for your OS → Install with default settings. Remember the password you set for the postgres user — you'll need it in Step 3.

⚠️  Write down your PostgreSQL password — you cannot recover it easily if forgotten.
📁

Copy the code folder to your computer

The ZIP file you downloaded contains the hangers-backend folder. Move it to a convenient location like your Desktop or Documents. Then open your terminal and navigate to it:

TERMINAL
cd Desktop/hangers-backend # or wherever you placed it npm install # installs all packages (~1 min)
2
Create the Database
Set up your PostgreSQL database for Hangers
🗄️

Create a new database called hangers_db

Open the PostgreSQL command line tool (psql) or use pgAdmin (installed with PostgreSQL). Run this command:

PSQL / PGADMIN SQL
CREATE DATABASE hangers_db;

If using psql, first connect with: psql -U postgres and enter your password.

3
Configure Your Environment File
Your personal settings — never share this file
📝

Create your .env file

In the hangers-backend folder, make a copy of .env.example and rename it to .env. Then fill in your values:

.env — FILL IN YOUR VALUES
PORT=5000 NODE_ENV=development # Replace 'yourpassword' with your PostgreSQL password DATABASE_URL="postgresql://postgres:yourpassword@localhost:5432/hangers_db" # Generate with: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))" JWT_SECRET="paste_your_generated_secret_here" JWT_CUSTOMER_EXPIRES_IN=30d JWT_STAFF_EXPIRES_IN=12h # Your MSG91 credentials (from msg91.com dashboard) MSG91_AUTH_KEY="your_msg91_key" MSG91_TEMPLATE_ID="your_template_id" # Razorpay test keys (from razorpay.com dashboard) RAZORPAY_KEY_ID="rzp_test_XXXXXXXXXX" RAZORPAY_KEY_SECRET="XXXXXXXXXXXXXXXX" SHOP_NAME="Hangers Clothes Spa" SHOP_PHONE="+91 7977417014"
ℹ️  MSG91 not set up yet? No problem — the server will print OTPs directly in the terminal window during development so you can still test everything.
🔑

Generate your JWT Secret

Run this in your terminal to generate a secure random secret:

TERMINAL
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

Copy the long string it outputs and paste it as your JWT_SECRET in the .env file.

4
Run the Database Migrations & Start Server
Create all tables and start your API
🔄

Run Prisma database migration

This creates all the tables in your hangers_db database automatically:

TERMINAL — in hangers-backend folder
npx prisma migrate dev --name init # ✅ Creates: customers, staff, addresses, sessions, otp_verifications, activity_logs
🌱

Seed the database — create your admin account

TERMINAL
node prisma/seed.js # Creates: admin@hangers.in / Hangers@2025 (change after first login!)
🚀

Start the development server

TERMINAL
npm run dev # You should see: ───────────────────────────────────────── 🧺 Hangers Clothes Spa ✅ Server running on port 5000 🌍 Environment: development ─────────────────────────────────────────
5
Test Your API Endpoints
Verify everything is working correctly
🩺

Health check — open this in your browser

BROWSER URL
http://localhost:5000/health
✅  You should see: {"success": true, "message": "🧺 Hangers API is running"}
🔌

Install a free API testing tool

Download Postman (getpostman.com) or Insomnia (insomnia.rest) — free tools to send API requests for testing. Then test these endpoints:

METHODENDPOINTWHAT IT DOES
GET/healthServer health check
POST/api/v1/auth/send-otpSend OTP to customer phone
POST/api/v1/auth/verify-otpVerify OTP → get token
GET/api/v1/auth/meGet customer profile (needs token)
POST/api/v1/staff/auth/loginStaff login → get token
GET/api/v1/staff/auth/meGet staff profile + permissions
👤

Test Staff Login

In Postman, create a POST request to http://localhost:5000/api/v1/staff/auth/login with this body:

REQUEST BODY (JSON)
{ "email": "admin@hangers.in", "password": "Hangers@2025" }
✅  You should get back a token and your staff profile with role SUPER_ADMIN and all permissions.
📱

Test Customer OTP Login

POST to /api/v1/auth/send-otp with body {"phone": "9876543210"}. The OTP will print in your terminal (dev mode). Then POST to /api/v1/auth/verify-otp:

REQUEST BODY (JSON)
{ "phone": "9876543210", "otp": "123456", // use the OTP printed in terminal "name": "Test Customer" }
